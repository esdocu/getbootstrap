Bootstrap incluye muchas [propiedades personalizadas de CSS (variables)](https://developer.mozilla.org/es/docs/Web/CSS/Using_CSS_custom_properties) en su CSS compilado para una personalización en tiempo real sin la necesidad de volver a compilar Sass. Estas proporcionan un acceso fácil a valores de uso común como nuestros colores de tema, puntos de interrupción y pilas de fuentes principales cuando trabajas en el inspector de tu navegador, un sandbox de código o un prototipado general.

**Todas nuestras propiedades personalizadas tienen el prefijo `bs-`** para evitar conflictos con CSS de terceros.

## Variables raíz

Aquí están las variables que incluimos (ten en cuenta que se requiere el `:root`) a las que se puede acceder en cualquier lugar donde se cargue el CSS de Bootstrap. Se encuentran en nuestro archivo `_root.scss` y se incluyen en nuestros archivos dist compilados.

### Por defecto

Estas variables CSS están disponibles en todas partes, independientemente del modo de color.

<Code lang="css" filePath="dist/css/bootstrap.css" fileMatch=":(root,\n\[data-bs-theme=light\] {[^}]*})" />

### Modo oscuro

Estas variables están limitadas a nuestro modo oscuro integrado.

<Code lang="css" filePath="dist/css/bootstrap.css" fileMatch="(\[data-bs-theme=dark\] {[^}]*})" />

## Variables de componentes

Bootstrap 5 está haciendo un uso cada vez mayor de propiedades personalizadas como variables locales para varios componentes. De esta manera reducimos nuestro CSS compilado, aseguramos que los estilos no se hereden en lugares como tablas anidadas y permitimos un rediseño y extensión básicos de los componentes de Bootstrap después de la compilación de Sass.

Echa un vistazo a nuestra documentación de tablas para obtener información sobre [cómo usamos las variables CSS]([[docsref:/content/tables#how-do-the-variants-and-accented-tables-work]]). Nuestras [barras de navegación también utilizan variables CSS]([[docsref:/components/navbar#css]]) a partir de la v5.2.0. También estamos usando variables CSS en nuestras cuadrículas (principalmente para los canales (gutters) en la [nueva cuadrícula CSS opcional]([[docsref:/layout/css-grid]])), y en el futuro se utilizarán más componentes.

Siempre que sea posible, asignaremos variables CSS al nivel de componente base (por ejemplo, `.navbar` para la barra de navegación y sus subcomponentes). Esto reduce las dudas sobre dónde y cómo personalizar, y permite modificaciones sencillas por parte de nuestro equipo en futuras actualizaciones.

## Prefijo

La mayoría de las variables CSS usan un prefijo para evitar colisiones con tu propio código base. Este prefijo se suma al `--` que se requiere en cada variable CSS.

Personaliza el prefijo a través de la variable Sass `$prefix`. Por defecto, está establecido en `bs-` (ten en cuenta el guión final).

## Ejemplos

Las variables CSS ofrecen una flexibilidad similar a las variables de Sass, pero sin la necesidad de compilación antes de ser servidas al navegador. Por ejemplo, aquí estamos restableciendo los estilos de fuente y enlace de nuestra página con variables CSS.

```css
body {
  font: 1rem/1.5 var(--bs-font-sans-serif);
}
a {
  color: var(--bs-blue);
}
```

## Puntos de interrupción de la cuadrícula

Si bien incluimos nuestros puntos de interrupción de cuadrícula como variables CSS (excepto para `xs`), ten en cuenta que **las variables CSS no funcionan en las consultas de medios (media queries)**. Esto es por diseño en la especificación CSS para variables, pero puede cambiar en los próximos años con el soporte para variables `env()`. Echa un vistazo a [esta respuesta de Stack Overflow](https://stackoverflow.com/a/47212942) para obtener algunos enlaces útiles. Mientras tanto, puedes usar estas variables en otras situaciones de CSS, así como en tu JavaScript.
