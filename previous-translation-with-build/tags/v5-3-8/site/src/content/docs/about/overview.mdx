## Equipo

Bootstrap es mantenido por un [pequeño equipo de desarrolladores](https://github.com/orgs/twbs/people) en GitHub. Buscamos activamente hacer crecer este equipo y nos encantaría saber de ti si te entusiasma el CSS a escala, escribir y mantener plugins de JavaScript vanilla y mejorar los procesos de herramientas de compilación para el código frontend.

## Historia

Creado originalmente por un diseñador y un desarrollador en Twitter, Bootstrap se ha convertido en uno de los frameworks front-end y proyectos de código abierto más populares del mundo.

Bootstrap fue creado en Twitter a mediados de 2010 por [@mdo](https://x.com/mdo) y [@fat](https://x.com/fat). Antes de ser un framework de código abierto, Bootstrap era conocido como _Twitter Blueprint_. A los pocos meses de desarrollo, Twitter celebró su [primera Hack Week](https://blog.x.com/engineering/en_us/a/2010/hack-week) y el proyecto explotó a medida que desarrolladores de todos los niveles se unieron sin ninguna guía externa. Sirvió como guía de estilo para el desarrollo de herramientas internas en la empresa durante más de un año antes de su lanzamiento público, y continúa haciéndolo hoy en día.

[Lanzado](https://blog.x.com/developer/en_us/a/2011/bootstrap-twitter) originalmente el <time datetime="2011-08-19 11:25">viernes 19 de agosto de 2011</time>, desde entonces hemos tenido más de [veinte lanzamientos]([[config:repo]]/releases), incluyendo dos importantes reescrituras con la v2 y la v3. Con Bootstrap 2, agregamos funcionalidad responsiva a todo el framework como una hoja de estilo opcional. Basándonos en eso, con Bootstrap 3, reescribimos la biblioteca una vez más para hacerla responsiva por defecto con un enfoque mobile first (primero para móviles).

Con Bootstrap 4, volvimos a reescribir el proyecto para tener en cuenta dos cambios arquitectónicos clave: una migración a Sass y el paso a flexbox de CSS. Nuestra intención es ayudar de una pequeña manera a hacer avanzar a la comunidad de desarrollo web presionando por propiedades CSS más nuevas, menos dependencias y nuevas tecnologías en navegadores más modernos.

Nuestro último lanzamiento, Bootstrap 5, se centra en mejorar la base de código de la v4 con la menor cantidad posible de cambios incompatibles importantes. Mejoramos las características y componentes existentes, eliminamos el soporte para navegadores más antiguos, descartamos jQuery a favor de JavaScript puro y adoptamos tecnologías más preparadas para el futuro como las propiedades personalizadas de CSS como parte de nuestras herramientas.
