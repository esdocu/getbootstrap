## CSS y JS compilados

Descarga el código compilado listo para usar de **Bootstrap v[[config:current_version]]** para agregarlo fácilmente a tu proyecto, el cual incluye:

- Paquetes CSS compilados y minificados (consulta la [comparación de archivos CSS]([[docsref:/getting-started/contents#archivos-css]]))
- Plugins JavaScript compilados y minificados (consulta la [comparación de archivos JS]([[docsref:/getting-started/contents#archivos-js]]))

Esto no incluye documentación, archivos fuente o dependencias de JavaScript opcionales como Popper.

## Archivos fuente

Compila Bootstrap con tu propia canalización de recursos (asset pipeline) descargando nuestros archivos fuente de Sass, JavaScript y documentación. Esta opción requiere algunas herramientas adicionales:

- [Compilador de Sass]([[docsref:/getting-started/contribute#sass]]) para compilar archivos fuente de Sass en archivos CSS
- [Autoprefixer](https://github.com/postcss/autoprefixer) para el prefijado de proveedores de CSS

En caso de que necesites nuestro conjunto completo de [herramientas de compilación]([[docsref:/getting-started/contribute#configuración-de-herramientas]]), se incluyen para desarrollar Bootstrap y su documentación, pero es probable que no sean adecuadas para tus propios fines.

## CDN a través de jsDelivr

Omite la descarga con [jsDelivr](https://www.jsdelivr.com/) para entregar la versión almacenada en caché del CSS y JS compilados de Bootstrap a tu proyecto.

```html
<link href="[[config:cdn.css]]" rel="stylesheet" integrity="[[config:cdn.css_hash]]" crossorigin="anonymous">
<script src="[[config:cdn.js_bundle]]" integrity="[[config:cdn.js_bundle_hash]]" crossorigin="anonymous"></script>
```

Si estás utilizando nuestro JavaScript compilado y prefieres incluir Popper por separado, agrega Popper antes de nuestro JS, preferiblemente a través de un CDN.

```html
<script src="[[config:cdn.popper]]" integrity="[[config:cdn.popper_hash]]" crossorigin="anonymous"></script>
<script src="[[config:cdn.js]]" integrity="[[config:cdn.js_hash]]" crossorigin="anonymous"></script>
```

### CDNs alternativos

Recomendamos [jsDelivr](https://www.jsdelivr.com/) y lo usamos nosotros mismos en nuestra documentación. Sin embargo, en algunos casos (como en algunos países o entornos específicos), es posible que debas usar otros proveedores de CDN como [cdnjs](https://cdnjs.com/) o [unpkg](https://unpkg.com/).

Encontrarás los mismos archivos en estos proveedores de CDN, aunque con diferentes URL. Con cdnjs, puedes [usar este enlace directo del paquete Bootstrap](https://cdnjs.com/libraries/bootstrap) para copiar y pegar fragmentos de HTML listos para usar para cada archivo de distribución de cualquier versión de Bootstrap.

<Callout type="warning">
**Si los hashes SRI difieren para un archivo determinado, no debes usar los archivos de ese CDN, porque significa que el archivo fue modificado por otra persona.**
</Callout>

Ten en cuenta que debes comparar hashes de la misma longitud, por ejemplo, `sha384` con `sha384`, de lo contrario es de esperar que sean diferentes.

Como tal, puedes usar una herramienta en línea como [SRI Hash Generator](https://www.srihash.org/) para asegurarte de que los hashes sean los mismos para un archivo determinado.

Alternativamente, asumiendo que tienes OpenSSL instalado, puedes lograr lo mismo desde la CLI, por ejemplo:

```sh
openssl dgst -sha384 -binary bootstrap.min.js | openssl base64 -A
```

## Gestores de paquetes

Integra los **archivos fuente** de Bootstrap en casi cualquier proyecto con algunos de los gestores de paquetes más populares. No importa el gestor de paquetes, Bootstrap **requerirá un [compilador de Sass]([[docsref:/getting-started/contribute#sass]]) y [Autoprefixer](https://github.com/postcss/autoprefixer)** para una configuración que coincida con nuestras versiones compiladas oficiales.

### npm

Instala Bootstrap en tus aplicaciones impulsadas por Node.js con el [paquete npm](https://www.npmjs.com/package/bootstrap):

```sh
npm install bootstrap@[[config:current_version]]
```

`const bootstrap = require('bootstrap')` o `import bootstrap from 'bootstrap'` cargarán todos los plugins de Bootstrap en un objeto `bootstrap`.
El módulo `bootstrap` en sí exporta todos nuestros plugins. Puedes cargar manualmente los plugins de Bootstrap individualmente cargando los archivos `/js/dist/*.js` en el directorio de nivel superior del paquete.

El archivo `package.json` de Bootstrap contiene algunos metadatos adicionales bajo las siguientes claves:

- `sass`: ruta al archivo fuente principal de [Sass](https://sass-lang.com/) de Bootstrap
- `style`: ruta al CSS no minificado de Bootstrap que se ha compilado utilizando la configuración predeterminada (sin personalización)

<Callout name="info-npm-starter" />

### yarn

Instala Bootstrap en tus aplicaciones impulsadas por Node.js con el [paquete yarn](https://yarnpkg.com/en/package/bootstrap):

```sh
yarn add bootstrap@[[config:current_version]]
```

<Callout type="warning">
**Yarn 2+ (también conocido como Yarn Berry) no admite el directorio `node_modules` de forma predeterminada**: el uso de nuestro [ejemplo de Sass y JS](https://github.com/twbs/examples/tree/main/sass-js) necesita algunos ajustes:

```sh
yarn config set nodeLinker node-modules # Usa el enlazador node_modules
touch yarn.lock # Crea un archivo yarn.lock vacío
yarn install # Instala las dependencias
yarn start # Inicia el proyecto
```
</Callout>

### Bun

Instala Bootstrap en tus aplicaciones impulsadas por Bun o Node.js con la [CLI de Bun](https://bun.sh/):

```sh
bun add bootstrap@[[config:current_version]]
```

### RubyGems

Instala Bootstrap en tus aplicaciones de Ruby usando [Bundler](https://bundler.io/) (**recomendado**) y [RubyGems](https://rubygems.org/) agregando la siguiente línea a tu [`Gemfile`](https://bundler.io/guides/gemfile.html):

```ruby
gem 'bootstrap', '~> [[config:current_ruby_version]]'
```

Alternativamente, si no estás usando Bundler, puedes instalar la gema ejecutando este comando:

```sh
gem install bootstrap -v [[config:current_ruby_version]]
```

[Consulta el README de la gema](https://github.com/twbs/bootstrap-rubygem/blob/main/README.md) para más detalles.

### Composer

También puedes instalar y administrar el Sass y JavaScript de Bootstrap usando [Composer](https://getcomposer.org/):

```sh
composer require twbs/bootstrap:[[config:current_version]]
```

### NuGet

Si desarrollas en .NET Framework, también puedes instalar y administrar el [CSS](https://www.nuget.org/packages/bootstrap/) o [Sass](https://www.nuget.org/packages/bootstrap.sass/) y JavaScript de Bootstrap usando [NuGet](https://www.nuget.org/). Los proyectos más nuevos deben usar [libman](https://learn.microsoft.com/en-us/aspnet/core/client-side/libman/) u otro método, ya que NuGet está diseñado para código compilado, no para recursos de frontend.

```powershell
Install-Package bootstrap
```

```powershell
Install-Package bootstrap.sass
```
