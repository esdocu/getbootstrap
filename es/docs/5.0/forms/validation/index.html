Validación

        

        
Proporciona feedback valioso y accionable a tus usuarios con la validación de formularios HTML5, mediante los comportamientos predeterminados del navegador o estilos personalizados y JavaScript.

Somos conscientes de que actualmente los estilos de validación personalizados del lado del cliente y los tooltips no son accesibles, ya que no se exponen a las tecnologías de asistencia. Mientras trabajamos en una solución, recomendaríamos usar la opción del lado del servidor o el método de validación predeterminado del navegador.



Cómo funciona


Así es como funciona la validación de formularios con Bootstrap:


    

  • La validación de formularios HTML se aplica mediante dos pseudo-clases de CSS, :invalid y :valid. Se aplica a los elementos <input>, <select> y <textarea>.
    • 

  • Bootstrap aplica los estilos :invalid y :valid a la clase padre .was-validated, usualmente aplicada al <form>. De lo contrario, cualquier campo requerido sin un valor se mostraría como inválido al cargar la página. De esta manera, puedes elegir cuándo activarlos (típicamente después de intentar el envío del formulario).
    • 

  • Para restablecer la apariencia del formulario (por ejemplo, en el caso de envíos de formulario dinámicos usando AJAX), elimina la clase .was-validated del <form> nuevamente después del envío.
    • 

  • Como alternativa, las clases .is-invalid y .is-valid pueden usarse en lugar de las pseudo-clases para la validación del lado del servidor. No requieren una clase padre .was-validated.
    • 

  • Debido a restricciones en cómo funciona CSS, no podemos (en la actualidad) aplicar estilos a un <label> que viene antes de un control de formulario en el DOM sin la ayuda de JavaScript personalizado.
    • 

  • Todos los navegadores modernos soportan la API de validación de restricciones, una serie de métodos de JavaScript para validar controles de formulario.
    • 

  • Los mensajes de feedback pueden utilizar los valores predeterminados del navegador (diferentes para cada navegador, y no estilizables vía CSS) o nuestros estilos de feedback personalizados con HTML y CSS adicionales.
    • 

  • Puedes proporcionar mensajes de validez personalizados con setCustomValidity en JavaScript.
    • 



Con eso en mente, considera las siguientes demos para nuestros estilos de validación de formularios personalizados, clases opcionales del lado del servidor y valores predeterminados del navegador.


Estilos personalizados


Para mensajes de validación de formulario personalizados de Bootstrap, necesitarás añadir el atributo booleano novalidate a tu <form>. Esto deshabilita los tooltips de feedback predeterminados del navegador, pero aún proporciona acceso a las APIs de validación de formulario en JavaScript. Intenta enviar el formulario a continuación; nuestro JavaScript interceptará el botón de envío y te transmitirá el feedback. Al intentar enviar, verás los estilos :invalid y :valid aplicados a tus controles de formulario.


Los estilos de feedback personalizados aplican colores, bordes, estilos de foco e iconos de fondo personalizados para comunicar mejor el feedback. Los iconos de fondo para <select>s solo están disponibles con .form-select, y no con .form-control.

// Example starter JavaScript for disabling form submissions if there are invalid fields
(function () {
  'use strict'

  // Fetch all the forms we want to apply custom Bootstrap validation styles to
  var forms = document.querySelectorAll('.needs-validation')

  // Loop over them and prevent submission
  Array.prototype.slice.call(forms)
    .forEach(function (form) {
      form.addEventListener('submit', function (event) {
        if (!form.checkValidity()) {
          event.preventDefault()
          event.stopPropagation()
        }

        form.classList.add('was-validated')
      }, false)
    })
})()


Valores predeterminados del navegador


¿No te interesan los mensajes de feedback de validación personalizados o escribir JavaScript para cambiar los comportamientos del formulario? No hay problema, puedes usar los valores predeterminados del navegador. Intenta enviar el formulario a continuación. Dependiendo de tu navegador y sistema operativo, verás un estilo de feedback ligeramente diferente.


Aunque estos estilos de feedback no se pueden estilizar con CSS, aún puedes personalizar el texto de feedback a través de JavaScript.

Lado del servidor


Recomendamos usar validación del lado del cliente, pero en caso de que requieras validación del lado del servidor, puedes indicar campos de formulario inválidos y válidos con .is-invalid y .is-valid. Ten en cuenta que .invalid-feedback también es compatible con estas clases.


Para campos inválidos, asegúrate de que el mensaje de feedback/error inválido esté asociado con el campo de formulario relevante usando aria-describedby (ten en cuenta que este atributo permite referenciar más de un id, en caso de que el campo ya apunte a texto de formulario adicional).


Para arreglar problemas con los radios de borde, los grupos de entrada requieren una clase .has-validation adicional.

Tooltips


Si el diseño de tu formulario lo permite, puedes intercambiar las clases .{valid|invalid}-feedback por las clases .{valid|invalid}-tooltip para mostrar el feedback de validación en un tooltip estilizado. Asegúrate de tener un padre con position: relative para el posicionamiento del tooltip. En el ejemplo a continuación, nuestras clases de columna ya tienen esto, pero tu proyecto puede requerir una configuración alternativa.

Mixins


Dos mixins se combinan, a través de nuestro loop, para generar nuestros estilos de feedback de validación de formulario.


@mixin form-validation-state-selector($state) {
  @if ($state == "valid" or $state == "invalid") {
    .was-validated #{if(&, "&", "")}:#{$state},
    #{if(&, "&", "")}.is-#{$state} {
      @content;
    }
  } @else {
    #{if(&, "&", "")}.is-#{$state} {
      @content;
    }
  }
}

Map


Este es el Sass map de validación de _variables.scss. Sobrescribe o extiende esto para generar estados diferentes o adicionales.


$form-validation-states: (
  "valid": (
    "color": $form-feedback-valid-color,
    "icon": $form-feedback-icon-valid
  ),
  "invalid": (
    "color": $form-feedback-invalid-color,
    "icon": $form-feedback-icon-invalid
  )
);



Los maps de $form-validation-states pueden contener tres parámetros opcionales para sobrescribir los tooltips y los estilos de foco.


Loop


Usado para iterar sobre los valores del Sass map $form-validation-states para generar nuestros estilos de validación. Cualquier modificación al Sass map anterior se reflejará en tu CSS compilado a través de este loop.


@each $state, $data in $form-validation-states {
  @include form-validation-state($state, $data...);
}



Personalización


Los estados de validación se pueden personalizar vía Sass con el Sass map $form-validation-states. Ubicado en nuestro archivo _variables.scss, este Sass map es cómo generamos los estados de validación valid/invalid predeterminados. Incluido hay un map anidado para personalizar el color, icono, color de tooltip y sombra de foco de cada estado. Aunque los navegadores no soportan otros estados, aquellos que usan estilos personalizados pueden añadir fácilmente feedback de formulario más complejo.


Ten en cuenta que no recomendamos personalizar los valores de $form-validation-states sin también modificar el mixin form-validation-state.
